listening to events

events are things that happen on discord, like a user joining your server or someone sending a message. clientcord lets you "hear" these events and run code in response.

Step 1: The Setup

make sure your cc_init.py tells clientcord where to find your event files. usually, we put them in an /events folder.

cc_init.py
# ... (other config)
events = "/events"

Step 2: Create an Event File

every event you want to handle should be its own file in /events. you use the @event decorator to tell clientcord which event the function is for.

When the Bot starts (READY)

events/ready.py
from clientcord.declare import event

@event("READY")
async def on_ready(payload):
    print("I'm awake!")

When someone speaks (MESSAGE_CREATE)

events/message.py
from clientcord.declare import event

@event("MESSAGE_CREATE")
async def on_message(payload):
    # 'payload' is just a dictionary with all the message data!
    content = payload.get("content", "")
    print(f"I heard: {content}")

example: guild member add event

events/member_join.py
from clientcord.declare import event

@event("GUILD_MEMBER_ADD")
async def on_member_join(payload):
    user = payload.get("user", {})
    print(f"{user.get('username')} joined the server")
events are auto-loaded from the /events folder. no manual registration needed.
