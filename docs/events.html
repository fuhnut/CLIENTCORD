listening to events

events let your bot react to things happening on discord, like messages being sent, members joining, or the bot going online.

step 1: update your config

make sure your cc_init.py has the events folder configured.

cc_init.py
from clientcord.intents import Intents

token = "your-token"
intents = Intents.default()
prefix = "!"

commands = "/commands"
components = "/components"
events = "/events"

step 2: create event files

each event file uses the @event decorator and an async function. place them inside the /events folder.

example: bot ready event

events/ready.py
from clientcord.declare import event

@event("READY")
async def on_ready(payload):
    print(f"bot is online")

example: message create event

events/message.py
from clientcord.declare import event

@event("MESSAGE_CREATE")
async def on_message(payload):
    content = payload.get("content", "")
    author = payload.get("author", {}).get("username", "unknown")
    print(f"{author}: {content}")

example: guild member add event

events/member_join.py
from clientcord.declare import event

@event("GUILD_MEMBER_ADD")
async def on_member_join(payload):
    user = payload.get("user", {})
    print(f"{user.get('username')} joined the server")
events are auto-loaded from the /events folder. no manual registration needed.
