webhooks

webhooks are a low-effort way to post messages to channels in discord. they do not require a bot account to post content, making them powerful for external applications.

creating a webhook

create a webhook in a guild channel cleanly.

creating via channel
webhook = await ctx.channel.create_webhook(name="GitHub Alerts")
print(f"Token: {webhook['token']}")
print(f"URL: {webhook['url']}")

executing a webhook

most applications execute a webhook directly using a tool, but you can native trigger webhooks using clientcord.

webhook execution
await ctx.client.execute_webhook(
    webhook_id="123456789",
    webhook_token="your_token_here",
    content="hello from webhook world!",
    username="Alert System", # Override name
    avatar_url="https://example.com/icon.png" # Override avatar profile
)
the execute method natively accepts components! simply pass components=[row] and clientcord translates the flags instantly.

managing webhooks

if you wish to edit a webhook's name or active channel.

editing webhook
await ctx.client.modify_webhook("webhook_id", {"name": "News Feed"})
deleting webhook
await ctx.client.delete_webhook("webhook_id", reason="no longer needed")

webhook models

  • id - the webhook id
  • type - type of webhook (Incoming, Channel Follower, Application)
  • guild_id - guild id where the webhook exists
  • channel_id - channel id mapped to
  • user - creator's user model
  • name - webhook display string
  • token - secure hook token
