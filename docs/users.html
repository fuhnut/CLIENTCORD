users

the User model represents a discord user globally natively across all contexts, whilst a Member represents a user mapped to a specific guild.

the user model

if you retrieve a user directly from the context, it exposes base properties:

fetching users
# Accessible directly from context for the command author
author = ctx.user

# Properties available over the User model:
print(author.id)            # Discord ID snowflake
print(author.username)      # Display username
print(author.avatar)        # Avatar hash
print(author.bot)           # Boolean if the user is a bot
print(author.system)        # Boolean if the user is an official discord system account

the member model

the Member model encapsulates the user within the current guild context and provides high-level functionality like assigning roles.

fetching a member
# Accessible directly from context!
member = ctx.member

print(member.nick)          # Guild nickname
print(member.roles)         # List of Role IDs
print(member.joined_at)     # Timestamp string
print(member.premium_since) # Server boosting since timestamp string
