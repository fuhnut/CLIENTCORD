messages

messages represent interactions in text, threads, or dms. clientcord abstracts the complexity with the Message model and the unified ctx.write() capability.

sending messages

whether you're working with prefix commands or application interactions, just use ctx.write(). it will automatically format a message or webhooks reply.

standard text response
await ctx.write("hello, world!")
replying with embeds and components
from clientcord.components import ActionRow, Button

# Components passed to write() are natively converted internally!
row = ActionRow().add_components(
    Button().set_label("Click").set_custom_id("btn")
)

await ctx.write(
    content="hello with a button!",
    components=[row]
)

ephemeral messages

ephemeral messages are visible only to the person who triggered the command interaction.

ephemeral response
# The 'flags' argument takes bitwise integers or list of string properties 
await ctx.write("this is a secret message only you can see!", flags=64)

managing existing messages

if you need to manipulate messages inside arbitrary channels, you can leverage the channel object.

editing / deleting
channel = await ctx.client.get_channel("123456789")

# Fetch target message
target_message_id = "987654321"

# Delete a message
await channel.delete_message(target_message_id)

# Edit a message
await channel.edit_message(target_message_id, content="new updated text")

message model properties

  • id - the message id
  • channel_id - the channel id where the message exists
  • author - the User model belonging to the sender
  • content - raw string content of the message
  • embeds - list of embed elements attached
