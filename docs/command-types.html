command types

clientcord commands execute as hybrid commands by default. this means your command can be invoked as an application command (slash) or through conventional plain-text messages (prefix).

controlling command routing

you can strictly enforce how a command is triggered using boolean flags inside the @command decorator. this is useful if you require ui components and solely want slash integration, or if you want a developer-only terminal command executed exclusively via prefix strings.

exclusive slash command
from clientcord.declare import command

# the slash=true flag blocks message_create events from ever reaching this command
@command(name="profile", description="view user profile", slash=True)
class ProfileCommand:
    async def run(self, ctx):
        await ctx.write("this command only triggers via slash commands!")
exclusive prefix command
# the prefix=true flag blocks interaction_create events and is hidden from discord's command picker ui
@command(name="eval", prefix=True)
class EvalCommand:
    async def run(self, ctx):
        await ctx.write("this command only triggers via traditional plain text prefix!")
explicit hybrid command
# the hybrid=true flag is the default out of the box option if no flags are provided
@command(name="ping", description="pong!", hybrid=True)
class PingCommand:
    async def run(self, ctx):
        await ctx.write("this command can be triggered cleanly via either method!")
