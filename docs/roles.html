roles

roles manage permissions, colors, and hierarchical access within a guild. clientcord allows you to manage roles easily, using native python arrays for setting detailed permissions.

creating roles

you can create roles using human-readable string literals for the permissions argument.

creating a new role
# The permissions argument fully supports an array of strings
await ctx.guild.create_role(
    name="Moderator",
    color=0x3498db, # Hexadecimal integer for colors
    hoist=True, # Display role separately in members list
    mentionable=True,
    permissions=["manage_messages", "kick_members", "ban_members"]
)

assigning roles

easily provide a role to a Member object from your command interactions.

assigning a role
target_member = ctx.guild.get_member("user_id")

# you can pass the role ID directly
await target_member.add_role("1234567890", reason="passed verification")

# or pass the human-readable role name (clientcord dynamically resolves it to the ID!)
await target_member.add_role("Moderator", reason="passed verification")
removing a role
await target_member.remove_role("Moderator", reason="punishment")

role properties

  • id - the role id
  • name - the role name
  • color - integer representation of hexadecimal code
  • hoist - if this role is pinned separately
  • permissions - the permission bit set
  • mentionable - whether the role is taggable
