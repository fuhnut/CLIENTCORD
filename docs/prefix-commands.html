prefix commands

clientcord handles prefix commands alongside slash commands. they share the same command classes and options system.

configuring the prefix

set the prefix field in your cc_init.py. you can use a single prefix or configure multiple.

cc_init.py
from clientcord.intents import Intents

token = "your-token"
intents = Intents.default()
prefix = "!"

commands = "/commands"
events = "/events"

multiple prefixes

to use multiple prefixes, pass a lambda to the client configuration that returns a list.

bot.py
from clientcord.client import Client

client = Client(commands={
    "prefix": lambda msg: ["!", "?", "."]
})

if __name__ == "__main__":
    client.run()

how prefix commands work

when a user sends a message starting with the configured prefix, clientcord:

  • strips the prefix from the message
  • parses the command name and arguments
  • looks up the matching command class
  • executes it with the same context system as slash commands

prefix command with options

options work the same way. users pass them after the command name separated by spaces.

terminal usage
!greet bobby
commands/greet.py
from clientcord.declare import command, options
from clientcord.commands.options import string_option

@options(
    name=string_option(description="who to greet", required=True)
)
@command(name="greet", description="greet someone")
class GreetCommand:
    async def run(self, ctx, name: str):
        await ctx.write(f"hello {name}!")
