emoji

custom emoji can be managed per-guild. they can be static or animated, and optionally restricted to specific roles.

listing emoji

example
emojis = await ctx.guild.list_emojis()
example
emoji = await ctx.guild.get_emoji("emoji_id")

creating emoji

requires the CREATE_GUILD_EXPRESSIONS permission. the image must be base64 encoded and under 256kb.

example
import base64

with open("emoji.png", "rb") as f:
    image_data = base64.b64encode(f.read()).decode("utf-8")

emoji = await ctx.guild.create_emoji({
    "name": "cool_emoji",
    "image": f"data:image/png;base64,{image_data}",
    "roles": []
}, reason="adding new emoji")

modifying emoji

example
await ctx.guild.modify_emoji("emoji_id", {
    "name": "renamed_emoji"
}, reason="better name")
example
await ctx.guild.modify_emoji("emoji_id", {
    "roles": ["role_id_1", "role_id_2"]
})

deleting emoji

example
await ctx.guild.delete_emoji("emoji_id", reason="cleanup")

application emoji

apps can own up to 2000 emoji that only the app can use. no USE_EXTERNAL_EMOJIS permission required.

list app emoji
result = await client.list_application_emojis()
emojis = result["items"]
get a single app emoji
emoji = await client.get_application_emoji("emoji_id")

emoji formats

emoji can be uploaded as jpeg, png, gif, webp, or avif. all emoji are served as webp for best performance. for animated emoji, use the .webp extension with ?animated=true.

premium emoji

emoji with subscription roles are considered premium emoji and count toward a separate limit of 25. an emoji cannot have both subscription and non-subscription roles.

emoji model

import
from clientcord.models import Emoji

the Emoji model fields:

  • id - emoji id (null for standard unicode emoji)
  • name - emoji name
  • roles - role ids that can use this emoji
  • user - user who created this emoji
  • require_colons - whether colons are required
  • managed - whether this emoji is managed by an integration
  • animated - whether this emoji is animated
  • available - whether this emoji can be used
