audit logs

audit logs record every administrative action in a guild. entries are stored for 45 days and require the VIEW_AUDIT_LOG permission to read.

fetching audit logs

use get_guild_audit_log to fetch entries. you can filter by user, action type, and paginate with before/after.

get recent audit log entries
logs = await ctx.guild.fetch_audit_logs()
filter by user and action
from clientcord.models import AuditLogEvent

# get only kick actions
logs = await ctx.guild.fetch_audit_logs(
    action_type=AuditLogEvent.member_kick,
    limit=10
)

audit log entry structure

each entry has these fields:

  • id - unique entry id
  • action_type - the event type (integer)
  • target_id - id of the affected entity
  • user_id - who performed the action
  • changes - list of what changed
  • options - extra info for certain events
  • reason - audit log reason (1-512 chars)

reading changes

the changes array contains objects with key, old_value, and new_value fields.

example
logs = await ctx.guild.fetch_audit_logs(
    user_id="123456789" # the user who performed the actions
)
entry in logs["audit_log_entries"]: for change in entry.get("changes", []): key = change["key"] old = change.get("old_value") new = change.get("new_value") print(f"{key}: {old} -> {new}")

audit log reason

when your bot performs an action, pass a reason parameter to include it in the audit log. this works on any endpoint that supports X-Audit-Log-Reason.

example
await ctx.channel.delete(reason="no longer needed")
await ctx.channel.edit({"name": "renamed"}, reason="cleanup")
await ctx.guild.create_emoji(payload, reason="new emoji")
