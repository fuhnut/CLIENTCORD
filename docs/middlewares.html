command middlewares

middlewares are functions that run before a command executes. use them for permission checks, logging, rate limiting, or anything that should gate command access.

creating a middleware

middlewares/logger.py
from clientcord.declare import middleware

@middleware("logger")
async def logger_middleware(ctx, next_mw, stop):
    print(f"{ctx.user.id} ran a command")
    await next_mw()

the middleware receives three arguments:

  • ctx - the command context
  • next_mw - call this to pass to the next middleware or the command
  • stop - call this with a reason to block the command

using a middleware on a command

commands/shutdown.py
from clientcord.declare import command, middleware

@middleware("is_developer")
async def is_dev(ctx, next_mw, stop):
    if ctx.user.id != "your-user-id":
        return stop("unauthorized")
    await next_mw()

@command(name="shutdown", description="shut down the bot")
@is_dev
class ShutdownCommand:
    async def run(self, ctx):
        await ctx.write("shutting down")

stopping a middleware with an error

when a middleware calls stop(), the command will not execute and the error is forwarded to the on_middleware_error handler on the command class.

commands/admin.py
from clientcord.declare import command, middleware

@middleware("admin_only")
async def admin_check(ctx, next_mw, stop):
    if not ctx.user.id == "admin-id":
        return stop("you are not an admin")
    await next_mw()

@command(name="nuke", description="delete all messages")
@admin_check
class NukeCommand:
    async def run(self, ctx):
        await ctx.write("nuking channel")

    async def on_middleware_error(self, ctx, error):
        await ctx.write(f"blocked: {error}")

passing data through middlewares

you can pass data from a middleware to the command by attaching it to the context.

middlewares/timer.py
import time
from clientcord.declare import middleware

@middleware("timer")
async def timer_middleware(ctx, next_mw, stop):
    ctx.start_time = time.time()
    await next_mw()
