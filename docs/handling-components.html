handling components

when a user clicks a button or picks something from a menu, discord sends an "interaction" to your bot. clientcord makes it easy to catch these and run code!

Step 1: The Setup

just like commands and events, we give components their own folder (usually /components). update your cc_init.py to tell clientcord where to look.

cc_init.py
# ... (other config)
components = "/components"

Step 2: Create a Handler

every button or menu has a custom_id. you create a class in /components that matches that ID.

components/hello_button.py
from clientcord.components import ComponentCommand
from clientcord.declare import component

@component(custom_id="hello-world")
class HelloWorldButton(ComponentCommand):
    async def run(self, ctx):
        await ctx.write("hello world!")
component handlers are auto-loaded from the /components folder. the decorator automatically routes interactions to the correct class based on the custom_id.

handling select menus

handling select menus works exactly the same way. you can access the selected values through the ctx.interaction object.

components/color_picker.py
from clientcord.components import ComponentCommand
from clientcord.declare import component

@component(custom_id="color_pick")
class ColorPicker(ComponentCommand):
    async def run(self, ctx):
        # access values from the raw interaction payload
        selected = ctx.interaction.get("data", {}).get("values", [])
        await ctx.write(f"you picked {selected[0]}")
