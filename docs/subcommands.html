sub commands

subcommands let you group related functionality under a single parent command. for example, /account create and /account delete.

creating subcommands

define each subcommand as its own class, then group them under a parent.

commands/account_create.py
from clientcord.declare import command

@command(name="create", description="create an account", parent="account")
class AccountCreate:
    async def run(self, ctx):
        await ctx.write("account created")
commands/account_delete.py
from clientcord.declare import command

@command(name="delete", description="delete your account", parent="account")
class AccountDelete:
    async def run(self, ctx):
        await ctx.write("account deleted")
the parent parameter in the @command decorator tells clientcord which parent command to group the subcommand under. discord will show these as /account create and /account delete.
