auto moderation

auto moderation lets each guild create rules that trigger based on message content or member profile. when triggered, rules automatically execute actions like blocking messages or timing out users.

trigger types

each rule watches for a specific type of content.

trigger types
from clientcord.models import TriggerType

TriggerType.keyword         # 1 - words from a custom list
TriggerType.spam            # 3 - generic spam detection
TriggerType.keyword_preset  # 4 - internal wordsets (profanity, slurs, etc)
TriggerType.mention_spam    # 5 - too many unique mentions
TriggerType.member_profile  # 6 - keywords in member profiles

action types

actions are what happens when a rule triggers.

action types
from clientcord.models import ActionType

ActionType.block_message             # 1 - block the message
ActionType.send_alert_message        # 2 - log to a channel
ActionType.timeout                   # 3 - timeout the user
ActionType.block_member_interaction  # 4 - block text/voice/interactions

listing rules

requires the MANAGE_GUILD permission.

example
rules = await ctx.guild.list_auto_moderation_rules()
example
rule = await ctx.guild.get_auto_moderation_rule("rule_id")

creating a rule

create a keyword filter that blocks messages containing specific words and sends an alert to a log channel.

example
rule = await ctx.guild.create_auto_moderation_rule(
    name="profanity filter",
    event_type="message_send",
    trigger_type="keyword",
    trigger_metadata={
        "keyword_filter": ["badword1", "badword2"]
    },
    actions=[
        {"type": "block_message"}
    ],
    enabled=True
)

creating a mention spam rule

example
rule = await ctx.guild.create_auto_moderation_rule(
    name="spam filter",
    event_type="message_send",
    trigger_type="spam",
    actions=[
        {
            "type": "timeout",
            "metadata": {"duration_seconds": 3600}
        }
    ],
    enabled=True
)

creating a preset rule

example
rule = await ctx.guild.create_auto_moderation_rule(
    name="preset filter",
    event_type="message_send",
    trigger_type="keyword_preset",
    trigger_metadata={
        "presets": [
            "profanity",
            "slurs"
        ],
        "allow_list": ["hell", "damn"]
    },
    actions=[
        {"type": "block_message"}
    ],
    enabled=True
)

modifying a rule

example
await ctx.guild.modify_auto_moderation_rule(
    "rule_id",
    enabled=False,
    actions=[
        {"type": "block_message"}
    ]
)

deleting a rule:

example
await ctx.guild.delete_auto_moderation_rule("rule_id", reason="no longer needed")

keyword matching

keywords support wildcard patterns with *:

  • cat* - prefix match (catches "catch", "catapult")
  • *cat - suffix match (catches "wildcat")
  • *cat* - anywhere match (catches "location")
  • cat - whole word match (only "cat")

all matches are case insensitive. each keyword can be up to 60 characters.

limits

  • keyword rules: max 6 per guild, 1000 keywords, 10 regex patterns
  • spam rules: max 1 per guild
  • keyword preset rules: max 1 per guild, 1000 allow list entries
  • mention spam rules: max 1 per guild, max 50 mention limit
  • member profile rules: max 1 per guild
  • exempt roles: max 20 per rule
  • exempt channels: max 50 per rule
