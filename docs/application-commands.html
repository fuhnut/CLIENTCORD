application commands

clientcord allows you to manage application commands globally or directly in a guild. these commands display directly in the user's interface, allowing inputs, sub-commands, and options.

fetching global commands

getting a list of all your global application commands is easy using the http client directly. 

from clientcord import Client

client = Client(token="bot_token")

async def get_commands():
    # Application ID is fetched seamlessly behind the scenes
    commands = await client.get_global_commands()
    print(commands)

bulk overwriting commands

to sync all of your application commands at once to discord, use the bulk overwrite method. 

async def sync_commands():
    synced = await client.sync_commands("123456789") # Assuming testing guild ID
    # Use client.sync_commands() internally since it builds the payload natively now
    print("synced global commands!")

guild application commands

guild commands update instantly compared to global ones. sync them the exact same way but supply the guild id! 

async def sync_guild_commands():
    guild_id = "123456789"
    
    synced = await client.sync_commands(guild_id)
    print("synced guild-specific commands!")

creating & deleting single commands

you can also manage commands one by one. 

async def manage_command():
    new_cmd = await client.create_global_command(
        name="hello",
        description="says hello",
        type=1
    )
    
    command_id = new_cmd["id"]
    
    await client.edit_global_command(
        command_id, 
        description="says hello loudly"
    )
    
    await client.delete_global_command(command_id)

defining permissions

with clientcord, you don't need to manually calculate bitwise integers to define required permissions. instead, you can simply pass an array of intuitive strings. this works directly in the decorator! 

from clientcord.declare import command

@command(
    name="ban",
    description="bans a user",
    default_member_permissions=["ban_members"],
    bot_permissions=["ban_members", "send_messages"]
)
class BanCommand:
    async def run(self, ctx):
        await ctx.write("user banned!")

command models

clientcord fully implements discord application command structures internally such as ApplicationCommand, ApplicationCommandOption, and ApplicationCommandOptionChoice using strict msgspec.Struct types for maximum performance.
