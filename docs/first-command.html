creating your first command

commands are how users interact with your bot. clientcord makes it super easy to create "Slash Commands" (the ones that pop up when you type /).

Step 1: Create the Command File

think of a command name, like /ping. create a file named ping.py inside your /commands folder.

commands/ping.py
from clientcord.declare import command

# The name is what users type, description is what they see
@command(name="ping", description="is the bot alive?")
class PingCommand:
    # This runs whenever someone uses the command
    async def run(self, ctx):
        # Sends a message back to the user
        await ctx.write("pong! 🏓")
Remember: because you set commands = "/commands" in your config, clientcord finds this file automatically!

Step 2: Telling Discord about the Command

discord needs to know your command exists. we do this by "syncing". the best way is to do it once when the bot starts up.

events/ready.py
from clientcord.declare import event
# We import the client we created in bot.py
from bot import client 

@event("READY")
async def on_ready(payload):
    print(f"Logged in as {payload['user']['username']}")
    
    # This 'pushes' your commands to discord
    await client.sync_commands()
    print("Commands synced!")

step 3: run your bot

terminal
python bot.py

when the bot starts, clientcord compiles execution plans for all commands, connects to discord, and sync_commands() pushes the slash command metadata API payload out. you should see the /ping command available on your server!

using options

commands can accept user inputs through options. use the @options decorator and option constructors.

commands/ping.py
from clientcord.declare import command, options
from clientcord.commands.options import boolean_option

@options(
    hide=boolean_option(description="hide the response")
)
@command(name="ping", description="show bot latency")
class PingCommand:
    async def run(self, ctx, hide: bool = False):
        await ctx.write("pong")
