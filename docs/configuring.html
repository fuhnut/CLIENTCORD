configuring clientcord

clientcord is set up to automatically load your commands, events, and components from directories you specify. it's like magic!

Step 1: Get your Bot Token

you can't start your bot without a secret token from discord. here's how to get it:

  1. Go to the Discord Developer Portal.
  2. Create a "New Application" and give it a name.
  3. Click Bot in the sidebar.
  4. Click Reset Token (or Copy) to get your secret token. Never share this with anyone!

Step 2: Create cc_init.py

this is your bot's brain. create a file called cc_init.py in your project root.

cc_init.py
from clientcord.intents import Intents

# Paste your token here!
token = "your-bot-token-here"

# This tells discord what your bot is allowed to see
intents = Intents.default()

# The prefix for old-school commands (like !ping)
prefix = "!"

# Where clientcord should look for your code
commands = "/commands"    # Folder for slash commands
components = "/components" # Folder for buttons/modals
events = "/events"       # Folder for events like join/leave
Pro-tip: every .py file you put in those folders will be loaded automatically. you don't even have to import them yourself!

Step 3: Create the Main File

now you need a way to actually start the bot. create bot.py.

bot.py
from clientcord.client import Client

# This looks for your cc_init.py automatically
client = Client()

if __name__ == "__main__":
    client.run()

lazy state & lmdb caching

clientcord is designed for extreme memory efficiency. unlike other libraries that store thousands of objects in your ram, clientcord uses lazy modeling and 100% persistent state caching.

  • low ram: stays under 50MB of RAM even in servers with thousands of members.
  • lmdb storage: guild and member data is stored in the .clientcord_cache folder on disk.
  • stateful: your bot "remembers" server state across restarts without making extra API calls.

the .clientcord_cache folder is created automatically in your project root. it acts as the bot's permanent memory, using high-speed memory-mapping (LMDB) to balance disk space and performance.
