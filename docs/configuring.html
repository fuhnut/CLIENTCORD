configuring clientcord

clientcord is set up to automatically load your commands, events, and components from directories you specify.

creating the config file

create a file called cc_init.py in the root of your project. this is where clientcord picks up your token, intents, prefix, and folder locations.

cc_init.py
from clientcord.intents import Intents

token = "your-bot-token-here"
intents = Intents.default()
prefix = "!"

commands = "/commands"
components = "/components"
events = "/events"
the commands, components, and events fields tell clientcord which folders to auto-load python files from at startup. every .py file inside those folders will be imported automatically.

selecting the bot type

clientcord supports the gateway (websocket) connection to discord. the gateway client handles all events emitted by discord such as MESSAGE_CREATE, INTERACTION_CREATE, GUILD_CREATE, READY, among others.

the available events depend on the intents configured in your cc_init.py.

intents

intents use python's IntFlag for type-safe bitwise operations. here are the available methods:

intents usage
from clientcord.intents import Intents

all_intents = Intents.all()

default_intents = Intents.default()

custom = Intents.guilds | Intents.guild_messages | Intents.message_content

creating the main file

create your main bot script. this is the entry point that starts the gateway connection and loads everything.

bot.py
from clientcord.client import Client

client = Client()

if __name__ == "__main__":
    client.run()

when you call client.run(), clientcord will:

  • read your cc_init.py config
  • auto-load all files from your /commands, /components, and /events folders
  • compile execution plans for all registered commands
  • connect to the discord gateway
